# Codex Config Switcher 1.1.0

这次版本把产品从“配置切换工作台”推进成了更适合首次分发和代理站点使用的桌面工具：

- 新增预设级站点门户登录，可在应用内接管登录态并查看余额、模型和 token 使用情况。
- 新增首次打开引导，首次启动时会明确说明每一步会做什么、不会做什么，减少误操作。
- 新增安装受阻说明，区分“Apple 无法校验 / unknown developer”和“应用已损坏”两类系统提示。
- 主窗口滚动结构优化，右侧总览区不再长期挤压编辑区可见高度。

## 安装包

- 文件：`Codex-Config-Switcher-1.1.0.dmg`
- 平台：macOS 13+
- SHA256：`0f8b93a40ab6fbd5823ee9c947f037c20a93719c45da278dbf1ccf089cce3dd8`

## 安装方式

1. 下载 `Codex-Config-Switcher-1.1.0.dmg`。
2. 打开镜像后，将 `Codex Config Switcher.app` 拖到 `Applications`。
3. 如果首次打开看到“Apple 无法校验是否含恶意软件”或“unknown developer”，先尝试打开一次，再到“系统设置 -> 隐私与安全性”里点击“仍要打开 / Open Anyway”。
4. 如果提示“应用已损坏 / modified or damaged”，请先重新下载并确认来源可信，再决定是否继续放行。

## 本次重点更新

- 站点账户能力：
  - 预设新增门户地址配置。
  - 应用内登录站点账户后，可查看余额、累计 token、请求量、模型使用分布和可用模型列表。
  - 登录态和账号概览缓存独立存储，不会进入模板和预设导出。
- 首次引导：
  - 新增 welcome / 文件路径 / 导入现状 / 应用联动 / 可选站点账户 / 完成 6 步流程。
  - 菜单栏在 onboarding 未完成前切换成最小入口，避免误操作 live 配置。
  - 设置页支持重新打开首次引导。
- 布局与体验：
  - 右侧主区改成整体滚动，编辑区获得更大的可视高度。
  - 登录站点窗口的配色和错误处理优化，减少误报“无法打开站点登录页”的情况。

## 说明

- 当前 Release 仍是本地构建的未 notarize 安装包。
- 为了彻底减少 macOS 首次打开的安全拦截，后续建议升级到 `Developer ID + notarization + stapling` 分发链路。
